# reegen - Generator

The generator introspects your database and generates schema files + CRUD routes. Runs as a local tool within the project.

```bash
bun generator/resource.ts all
bun generator/schema.ts authors --force
bun generator/crud.ts countries
```

Supports MySQL and SQLite. See [generator/AGENTS.md](AGENTS.md) and [internals/GENERATOR_INTERNALS.md](../internals/GENERATOR_INTERNALS.md#generators) for the full documentation including:

- Common flags (`--force`, `--translate`, `--prefix`, `--parent`, `--pagination`)
- reeman menu options and bulk CRUD flow
- Nested CRUD (master-detail)
- Cursor & offset pagination (`--pagination cursor|offset`)
- Tags translation
- Column comment-driven field types
- Fulltext search
- Currency field rendering
- Placeholder naming convention
- `config/db_structure.ts` constants reference
- URL Localization via `route_name` keys

## Generated folder structure

```
routes/<table>/
├-- schema/                  # Generated by `bun generator/schema`
│   ├-- table.generated.ts   # Auto-generated (safe to overwrite)
│   ├-- table.ts             # User-editable, created once (never overwritten)
│   └-- validation-server.ts # Auto-generated Zod validation (safe to overwrite)
├-- translations/            # Generated DB translation keys (see below)
├-- index.ts                 # CRUD route handlers (generated by `bun generator/crud`)
├-- sql.ts                   # SQL query functions (generated by `bun generator/crud`)
├-- sql_view.ts              # View-based queries, if a view exists (generated by `bun generator/crud`)
├-- form.ree                 # Create/edit form template (generated by `bun generator/crud`)
└-- index.ree                # List/index template (generated by `bun generator/crud`)
```

## Nested child structure

When `--parent <table>` is specified, the child is placed directly under the parent's directory with a minimal file set:

```
routes/<parent>/
  <child>/
    index.ts           # Only create, edit, edit-data (JSON), validate
    sql.ts             # Scoped queries: all accept parent_id, filter by FK
    schema/
      table.ts         # Field definitions + parent export
      table.generated.ts
      validation_server.ts
    translations/            # DB translation keys (generated by sync_translations)
```

The child has **no `form.ree` or `index.ree`** - the form UI lives in the parent's `form.ree` inside `<!-- crud:children:start -->` markers.

### Examples

```sh
# Generate parent (standalone CRUD)
bun generator/resource.ts equipment

# Generate child nested under parent
bun generator/resource.ts equipment_items --parent equipment

# With prefix
bun generator/resource.ts equipment_items --parent equipment --prefix admin

# Refresh a child (preserves --parent)
bun generator/crud.ts equipment_items --parent equipment --refresh-fields
```

## Sync Translations

Translations are **DB-only** - the `translations` table is the single source of truth. No JSON files are used. To add or change translations:

- `UPDATE translations SET translation = ...` or `INSERT INTO translations (...) VALUES (...)` - direct DB edits
- `bun run sync:languages` - AI-powered sync (scans DB, translates missing keys, writes back to DB)
- `/system/translations` admin UI - manual editing through the app

```bash
bun generator/sync_translations.ts --translate   # translate missing keys via OpenRouter
```

After sync, the server's `POST /__reload-translations` endpoint (called by the generator) refreshes in-memory translation maps on the running server.

## DDL Cache

The generator uses a **DDL cache** (`generator/ddl_cache.ts`) to avoid repeated database introspection. On first run (reeman or CLI), it introspects the full database schema, detects foreign key relationships (native DDL, implicit `*_id` naming, and view JOIN patterns), and writes the result to `generator/ddl_cache.json`. Subsequent operations during the same session read from the cached JSON instead of re-querying the database.

The cache auto-expires after 24 hours. To force a refresh, use the reeman's "Rescan DDL Cache" option, or call `invalidate_cache()` from `generator/ddl_cache`.

## Auto-formatting

The CRUD generator runs `reettier` on the generated route directory automatically after all files are written.
